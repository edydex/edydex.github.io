General Concepts

Using Cisco IOS

Command Modes Overview

Cisco IOS has hierarchical command modes. Each mode has specific commands available and is identified by the prompt.

User EXEC Mode (Router>)
Limited read-only access - first mode after login. Can't view full config or make changes.
enable
Command to enter privileged EXEC mode from user mode
Privileged EXEC Mode (Router#)
Full read access - can view configs, run show commands, and access config mode. The # symbol indicates privileged access.
configure terminal
Enters global configuration mode - where most configuration happens
Global Config Mode (Router(config)#)
Used to configure device-wide settings like hostname, passwords, routing protocols
interface gigabitEthernet 0/0
Enters interface configuration mode for a specific interface
Interface Config Mode (Router(config-if)#)
Configure specific interface settings like IP address, description, speed/duplex

Navigation Between Modes

exit
Moves back one level (e.g., from config-if to config, from config to privileged EXEC)
end
Returns directly to privileged EXEC mode from any config mode - same as pressing Ctrl+Z
disable
Returns from privileged EXEC to user EXEC mode

Subnetting & CIDR

What is Subnetting?

Subnetting divides a larger network into smaller subnetworks. This conserves IP addresses, improves security by isolating network segments, and reduces broadcast traffic. CIDR (Classless Inter-Domain Routing) notation like /24 replaces the old subnet mask notation.

CIDR Notation Basics

The /number represents how many bits are the "network" portion (left to right). The remaining bits are for hosts.

192.168.1.0/24
24 network bits = 255.255.255.0 subnet mask. The /24 means first 3 octets are network, last octet is for hosts (254 usable IPs)
10.0.0.0/8
8 network bits = 255.0.0.0 subnet mask. Huge network with ~16 million IPs. First octet is network, last 3 octets for hosts.
172.16.0.0/16
16 network bits = 255.255.0.0. Common for medium-large networks (~65,000 IPs). First 2 octets are network.
192.168.1.128/25
25 network bits = 255.255.255.128. Splits a /24 network in half - each subnet has 126 usable IPs

Calculating Subnets - The Binary Method

To find network address, broadcast address, and usable range, you need to understand binary. Each octet is 8 bits.

Example: Find subnet details for 192.168.1.75/26
Step 1: Identify the subnet mask
/26 = 255.255.255.192 (first 26 bits are network, last 6 bits are hosts)
Step 2: Calculate the "magic number"
256 - 192 = 64. This is the subnet increment. Subnets start at 0, 64, 128, 192 in the last octet.
Step 3: Find which subnet contains .75
Subnets are: .0-.63, .64-.127, .128-.191, .192-.255. The IP .75 falls in the .64 subnet.
Network Address: 192.168.1.64
First address in the subnet - cannot be assigned to hosts (identifies the subnet itself)
Broadcast Address: 192.168.1.127
Last address in subnet (next subnet - 1) - used to send to all hosts in subnet, cannot be assigned
Usable Range: 192.168.1.65 - 192.168.1.126
62 usable IPs (64 total - 2 reserved). First host is network+1, last host is broadcast-1

Quick Subnetting Tricks

Number of Subnets = 2^(borrowed bits)
If you borrow 3 bits from host portion, you get 2^3 = 8 subnets
Hosts per Subnet = 2^(host bits) - 2
Subtract 2 for network and broadcast addresses. /26 has 6 host bits: 2^6 - 2 = 62 usable hosts
Magic Number = 256 - Subnet Mask Octet
For /26 (255.255.255.192): 256-192=64. Subnets increment by 64 in the interesting octet.

VLSM (Variable Length Subnet Masking)

VLSM allows using different subnet masks within the same network to efficiently allocate IP addresses based on needs.

Example: Divide 10.1.1.0/24 for 3 departments: Sales (100 hosts), HR (50 hosts), IT (25 hosts), plus 2 point-to-point links
Sales: 10.1.1.0/25
126 usable hosts (.1-.126). Use /25 (255.255.255.128) for largest department first
HR: 10.1.1.128/26
62 usable hosts (.129-.190). Use /26 (255.255.255.192) from remaining space
IT: 10.1.1.192/27
30 usable hosts (.193-.222). Use /27 (255.255.255.224) for smaller group
Link 1: 10.1.1.224/30
2 usable hosts (.225-.226). /30 is standard for point-to-point WAN links
Link 2: 10.1.1.228/30
2 usable hosts (.229-.230). Still have 10.1.1.232-255 available for future growth

Exam Tip: Always allocate largest subnets first when doing VLSM. Know your powers of 2 (2, 4, 8, 16, 32, 64, 128, 256) and common masks (/24, /26, /27, /28, /30) by heart. Practice finding the "interesting octet" quickly.

VLANs & Trunking

What are VLANs?

VLANs (Virtual LANs) logically segment a physical switch into multiple broadcast domains. Devices in different VLANs cannot communicate directly without a router. VLANs improve security, reduce broadcast traffic, and organize networks by function (Sales, IT, Guest, etc.) rather than physical location.

VLAN Types

Data VLAN (VLANs 1-1005)
Normal range VLANs for user traffic - most common type. VLAN 1 is the default VLAN.
Native VLAN (Default: VLAN 1)
Untagged traffic on trunk ports belongs to native VLAN. Change from VLAN 1 for security. Must match on both ends of trunk.
Management VLAN
VLAN for switch management (SSH, Telnet). Assign IP to SVI (Switch Virtual Interface) for remote access.
Voice VLAN
Special VLAN for VoIP phones - QoS priority for voice traffic. Separate from data VLAN on same port.

Creating & Configuring VLANs

Create VLAN 10 for Sales department and assign ports
Switch(config)# vlan 10
Creates VLAN 10 and enters VLAN configuration mode
Switch(config-vlan)# name Sales
Names the VLAN for easier identification - optional but recommended

Now assign interfaces to the VLAN:

Switch(config)# interface range fa0/1-10
Selects multiple interfaces at once - more efficient than configuring one by one
Switch(config-if-range)# switchport mode access
Sets ports as access ports (connects to end devices). Required before assigning VLAN.
Switch(config-if-range)# switchport access vlan 10
Assigns these ports to VLAN 10. All devices connected to these ports are now in Sales VLAN.

Trunk Ports & 802.1Q

Trunk ports carry traffic for multiple VLANs between switches (or switch-to-router). 802.1Q adds a 4-byte tag to frames to identify which VLAN they belong to.

Configure trunk between two switches
Switch(config)# interface gi0/1
Enter the interface connecting to another switch - typically GigabitEthernet or higher
Switch(config-if)# switchport mode trunk
Sets port to trunk mode permanently. Use 'switchport mode dynamic desirable' for DTP negotiation (not recommended).
Switch(config-if)# switchport trunk native vlan 99
Changes native VLAN from default VLAN 1 to VLAN 99 for security. Must match on both ends!
Switch(config-if)# switchport trunk allowed vlan 10,20,30,99
Limits which VLANs can traverse trunk - security best practice. Only allow necessary VLANs.

Managing Allowed VLANs on Trunks

switchport trunk allowed vlan all
Allows all VLANs (1-4094) on trunk - default setting but not recommended for security
switchport trunk allowed vlan add 40
Adds VLAN 40 to existing allowed list without removing others
switchport trunk allowed vlan remove 20
Removes VLAN 20 from allowed list while keeping others
switchport trunk allowed vlan except 1-10
Allows all VLANs except 1-10. Useful for blocking ranges.

Switch Virtual Interface (SVI) for Management

Layer 2 switches need an IP address for remote management (SSH/Telnet). Assign IP to a VLAN interface (SVI).

Switch(config)# interface vlan 99
Creates SVI for VLAN 99 (management VLAN). The VLAN must exist first.
Switch(config-if)# ip address 192.168.99.2 255.255.255.0
Assigns IP to switch for management access
Switch(config-if)# no shutdown
Enables the SVI - must have at least one active port in this VLAN
Switch(config)# ip default-gateway 192.168.99.1
Sets default gateway for switch to reach other networks. Only needed for Layer 2 switches.

Inter-VLAN Routing (Router on a Stick)

Why Inter-VLAN Routing?

Devices in different VLANs cannot communicate by default - they're in separate broadcast domains. Inter-VLAN routing uses a router (or Layer 3 switch) to forward traffic between VLANs. Router on a Stick uses a single physical router interface with multiple subinterfaces, each handling a different VLAN.

How It Works

The router interface connects to a switch trunk port. Each VLAN gets a subinterface on the router with 802.1Q encapsulation. The router acts as the default gateway for each VLAN, routing packets between them.

Topology:
PC1 (VLAN 10) → Switch Fa0/1 (access)
PC2 (VLAN 20) → Switch Fa0/2 (access)
Switch Gi0/1 (trunk) → Router Gi0/0 (subinterfaces)

Step 1: Configure Switch Trunk Port

Switch(config)# interface gi0/1
Enter interface connecting to router
Switch(config-if)# switchport mode trunk
Set port to trunk mode to carry multiple VLANs to router
Switch(config-if)# switchport trunk allowed vlan 10,20,99
Only allow necessary VLANs on trunk for security

Step 2: Configure Router Subinterfaces

Create a subinterface for each VLAN. Subinterface number typically matches VLAN ID for clarity (e.g., gi0/0.10 for VLAN 10).

Configure routing for VLAN 10 (192.168.10.0/24) and VLAN 20 (192.168.20.0/24)
Router(config)# interface gi0/0
Enter the physical interface connected to switch trunk
Router(config-if)# no shutdown
Enable physical interface - ALL subinterfaces depend on this being up
Router(config-if)# exit
Return to global config to create subinterfaces

Subinterface for VLAN 10

Router(config)# interface gi0/0.10
Creates subinterface .10 - notation is interface.subinterface-number
Router(config-subif)# encapsulation dot1q 10
Configures 802.1Q tagging for VLAN 10. MUST be configured before IP address.
Router(config-subif)# ip address 192.168.10.1 255.255.255.0
This IP becomes the default gateway for all devices in VLAN 10

Subinterface for VLAN 20

Router(config)# interface gi0/0.20
Creates subinterface for VLAN 20
Router(config-subif)# encapsulation dot1q 20
Tags traffic with VLAN 20. The number must match the VLAN on the switch.
Router(config-subif)# ip address 192.168.20.1 255.255.255.0
Default gateway for VLAN 20 devices

Handling Native VLAN

If you need to route the native VLAN (e.g., VLAN 99), add the native keyword to encapsulation command.

Router(config)# interface gi0/0.99
Subinterface for native VLAN 99 (management VLAN)
Router(config-subif)# encapsulation dot1q 99 native
The 'native' keyword tells router this is untagged traffic. Must match switch native VLAN.
Router(config-subif)# ip address 192.168.99.1 255.255.255.0
Gateway for management VLAN

Step 3: Configure End Devices

Set each PC's default gateway to the router's subinterface IP for its VLAN.

PC1 in VLAN 10
IP: 192.168.10.10/24, Gateway: 192.168.10.1 (router subinterface)
PC2 in VLAN 20
IP: 192.168.20.10/24, Gateway: 192.168.20.1 (router subinterface)

Static Routing

What is Static Routing?

Static routes are manually configured routes that tell a router how to reach specific networks. Unlike dynamic routing protocols (OSPF, EIGRP), static routes don't automatically adjust to topology changes. They're ideal for small networks, stub networks, or default routes to ISPs.

DHCP

What is DHCP?

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses and network configuration to devices. Instead of manually configuring every computer, DHCP centrally manages IP allocation from a pool of addresses. This saves time, prevents IP conflicts, and simplifies network management.

DHCP Process (DORA)

1. Discover (Broadcast)
Client broadcasts "I need an IP!" (255.255.255.255) because it has no IP yet
2. Offer (Unicast/Broadcast)
DHCP server replies "Here's an available IP for you" along with subnet mask, gateway, DNS
3. Request (Broadcast)
Client broadcasts "I accept this IP" (other DHCP servers see this and release their offers)

Basic Static Route Syntax

ip route <destination-network> <subnet-mask> <next-hop-ip | exit-interface> [AD]

Example Topology

Network A: 10.1.1.0/24 — Router1 (.1) [Gi0/0] ↔ [Gi0/0] (.2) Router2 — Network B: 10.2.2.0/24
WAN link between routers: 192.168.1.0/30

Method 1: Next-Hop IP Address

Specifies the IP of the next router in the path. Router performs recursive lookup to find exit interface.

On Router1: Configure route to Network B (10.2.2.0/24) via Router2
Router1(config)# ip route 10.2.2.0 255.255.255.0 192.168.1.2
Sends packets for 10.2.2.0/24 to next-hop 192.168.1.2 (Router2's interface)
On Router2: Configure route to Network A (10.1.1.0/24) via Router1
Router2(config)# ip route 10.1.1.0 255.255.255.0 192.168.1.1
Sends packets for 10.1.1.0/24 to next-hop 192.168.1.1 (Router1's interface)

Method 2: Exit Interface

Specifies which local interface to use. Best for point-to-point links (Serial, PPP). Faster lookup than next-hop method.

Router1(config)# ip route 10.2.2.0 255.255.255.0 gi0/0
Sends packets out Gi0/0 interface. Router assumes destination is directly connected on that link.

Method 3: Fully Specified (Next-Hop + Exit Interface)

Combines both methods. Required for multi-access networks (Ethernet). Most reliable but longest to type.

Router1(config)# ip route 10.2.2.0 255.255.255.0 gi0/0 192.168.1.2
Exit out Gi0/0 AND forward to 192.168.1.2. Best practice for Ethernet links.

A catch-all route for any destination not in the routing table. Commonly used to route to the Internet via ISP.

Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
0.0.0.0 0.0.0.0 means "all networks" - matches any destination. Send everything unknown to ISP.
Router(config)# ip route 0.0.0.0 0.0.0.0 gi0/1
Default route using exit interface - send all unknown traffic out Gi0/1

Floating Static Route (Backup Route)

A backup route with higher AD that only activates if primary route fails. Useful for redundancy.

Router(config)# ip route 10.2.2.0 255.255.255.0 192.168.1.2
Primary route (AD=1 by default)
Router(config)# ip route 10.2.2.0 255.255.255.0 192.168.2.2 5
Backup route (AD=5). Only used if primary route (AD=1) goes down. Higher AD = less preferred.

Summary/Aggregate Route

One route summarizing multiple networks - reduces routing table size.

Instead of four /24 routes (172.16.0.0, 172.16.1.0, 172.16.2.0, 172.16.3.0), use one summary:
Router(config)# ip route 172.16.0.0 255.255.252.0 10.1.1.2
172.16.0.0/22 covers 172.16.0.0 through 172.16.3.255 - single route for 4 /24 networks

IPv6 Static Routes

Same concept as IPv4, but uses IPv6 addresses and prefix length notation.

Router(config)# ipv6 unicast-routing
Must enable IPv6 routing first - off by default
Router(config)# ipv6 route 2001:db8:2::/64 2001:db8:1::2
IPv6 static route using next-hop address
Router(config)# ipv6 route ::/0 gi0/1
IPv6 default route - ::/0 is equivalent to 0.0.0.0/0 in IPv4
Router(config)# ipv6 route 2001:db8:2::/64 gi0/0 fe80::2
Fully specified IPv6 route - exit interface + link-local next-hop. Required for Ethernet.

What is DHCP?

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses and network configuration to devices. Port Security

What is Port Security?

Port security restricts which devices can connect to a switch port based on MAC addresses. This prevents unauthorized devices from accessing the network and mitigates MAC flooding attacks. You can limit the number of MACs allowed, specify which MACs are permitted, and define actions when violations occur.

Why Use Port Security?

Prevent Unauthorized Access
Only allowed devices (by MAC) can connect to the port
MAC Flooding Protection
Prevents attackers from flooding MAC table with fake addresses
DHCP Starvation Defense
Limits devices that can request DHCP addresses on a port
Compliance & Auditing
Track which devices connect to which ports for security policies

Basic Port Security Configuration

Enable port security on interface to allow only 1 device
Switch(config)# interface fa0/1
Enter the interface you want to secure
Switch(config-if)# switchport mode access
Port security ONLY works on access ports, not trunks. This command is mandatory.
Switch(config-if)# switchport port-security
Enables port security on the interface. Default: allows 1 MAC, violation mode is shutdown.

Maximum MAC Addresses

Control how many devices can connect to the port simultaneously.

Switch(config-if)# switchport port-security maximum 1
Default - allows only 1 MAC address on this port
Switch(config-if)# switchport port-security maximum 3
Allows up to 3 devices on this port (useful for IP phone + PC + one more device)

MAC Address Learning Methods

1. Dynamic Learning (Default)

switchport port-security
By default, MACs are learned dynamically but NOT saved to config. Lost on reload or interface down.

2. Static MAC Address

Switch(config-if)# switchport port-security mac-address 1234.5678.90ab
Manually specify allowed MAC address. Saved to config. Use for critical devices like servers.
Switch(config-if)# switchport port-security mac-address 0011.2233.4455
Can add multiple static MACs (up to maximum configured)

3. Sticky Learning (Recommended)

Switch(config-if)# switchport port-security mac-address sticky
Dynamically learns MACs and automatically adds them to running-config. Survives reload if config is saved.

When a device connects, its MAC is learned and added as: switchport port-security mac-address sticky 1234.5678.90ab

Violation Modes

Defines what happens when an unauthorized MAC tries to access the port.

switchport port-security violation shutdown
DEFAULT - Disables port (err-disabled), sends SNMP trap, increments violation counter. Port stays down until manually recovered.
switchport port-security violation restrict
Drops packets from unauthorized MACs, sends SNMP trap, increments violation counter. Port stays up for authorized MACs.
switchport port-security violation protect
Silently drops packets from unauthorized MACs. NO trap, NO violation counter increment. Port stays up.

Complete Configuration Examples

Example 1: Secure server port with specific MAC, shutdown on violation
interface fa0/5
switchport mode access
switchport port-security
switchport port-security mac-address aaaa.bbbb.cccc
switchport port-security violation shutdown
Server port secured with static MAC. Any other device causes port to shut down.
Example 2: Allow 2 devices, remember them, restrict violations
interface fa0/10
switchport mode access
switchport port-security
switchport port-security maximum 2
switchport port-security mac-address sticky
switchport port-security violation restrict
Allows first 2 devices, remembers them. 3rd device's traffic dropped, violation logged.
Example 3: IP Phone + PC (typical office setup)
interface fa0/15
switchport mode access
switchport access vlan 10
switchport voice vlan 20
switchport port-security
switchport port-security maximum 3
switchport port-security mac-address sticky
switchport port-security violation restrict
Allows phone + PC + 1 more device. Common for desk with IP phone daisy-chaining PC.

Recovering from err-disabled State

When violation mode is shutdown, port enters err-disabled state and must be manually recovered.

Switch(config)# interface fa0/1
Enter the err-disabled interface
Switch(config-if)# shutdown
First, administratively shut down the port
Switch(config-if)# no shutdown
Then bring it back up. This clears the err-disabled state.

Enables automatic recovery from port security violations
Switch(config)# errdisable recovery interval 300
Sets recovery timer to 300 seconds (default is 300). Port automatically re-enables after timer expires.

ACLs (Standard & Extended)

What are ACLs?

Access Control Lists (ACLs) are packet filters that permit or deny traffic based on criteria like source/destination IP, protocol, port numbers. ACLs are processed top-to-bottom, stopping at the first match. They're used for security (blocking unwanted traffic), QoS, NAT definition, and more.

ACL Key Concepts

Top-to-Bottom Processing
ACL checks each line sequentially. First match wins - stops processing remaining lines.
Implicit Deny All
At the end of every ACL is an invisible "deny any" statement. If no permit match, packet is dropped.
One ACL per Interface per Direction
Each interface can have one inbound and one outbound ACL. Inbound filters before routing, outbound after.
Inbound vs Outbound
Inbound: Checked as packet enters interface. Outbound: Checked as packet exits interface.

Standard ACLs (1-99, 1300-1999)

Filter based on source IP address only. Simple but limited. Apply close to destination.

Standard ACL Syntax

access-list <1-99> {permit|deny} <source> [wildcard-mask]
Example: Deny host 192.168.10.5, permit all others
Router(config)# access-list 10 deny 192.168.10.5 0.0.0.0
Denies specific host. Wildcard 0.0.0.0 means exact match (can also use keyword "host 192.168.10.5")
Router(config)# access-list 10 permit any
Permits all other traffic. Without this, implicit deny blocks everything else!
Router(config)# interface gi0/0
Enter interface where ACL will be applied
Router(config-if)# ip access-group 10 out
Applies ACL 10 to outbound traffic on this interface. Use 'in' for inbound filtering.

NAT/PAT

What is NAT?

Network Address Translation (NAT) translates private IP addresses to public IP addresses, allowing internal devices to access the Internet. It conserves public IPs (billions of devices share limited IPv4 space) and provides security by hiding internal network structure. PAT (Port Address Translation) is NAT overload - many private IPs share one public IP.

NAT Terminology

Inside Local
Private IP address assigned to host on internal network (e.g., 192.168.1.10)
Inside Global
Public IP address representing inside device to outside world (e.g., 203.0.113.5)
Outside Local
IP address of outside host as it appears to inside network (usually same as outside global)
Outside Global
Public IP address assigned to host on external network (e.g., 8.8.8.8)

NAT Types

Static NAT (One-to-One)
Maps one private IP to one public IP permanently. Used for servers that need consistent external IP.
Dynamic NAT (Many-to-Many)
Maps private IPs to pool of public IPs dynamically. First-come-first-served. If pool exhausted, no translation.
PAT / NAT Overload (Many-to-One)
Maps many private IPs to single public IP using port numbers. Most common type (home routers use this).

Interface Designation

Must designate inside and outside interfaces for NAT to work.

Router(config)# interface gi0/0
Interface facing internal network (LAN)
Router(config-if)# ip nat inside
Marks interface as NAT inside - packets FROM this interface will be translated
Router(config)# interface gi0/1
Interface facing Internet/ISP (WAN)
Router(config-if)# ip nat outside
Marks interface as NAT outside - packets TO this interface use translated addresses

Static NAT Configuration

Map internal web server 192.168.1.10 to public IP 203.0.113.5
Router(config)# ip nat inside source static 192.168.1.10 203.0.113.5
Creates permanent one-to-one mapping. External users access 203.0.113.5, router forwards to 192.168.1.10

This creates bidirectional translation - inbound and outbound traffic automatically translated.

Multiple Static Mappings

ip nat inside source static 192.168.1.10 203.0.113.5
Web server
ip nat inside source static 192.168.1.20 203.0.113.6
Mail server
ip nat inside source static 192.168.1.30 203.0.113.7
FTP server

Dynamic NAT Configuration

Translate internal 192.168.10.0/24 network using pool of public IPs 203.0.113.10-14

Step 1: Define Inside Addresses (ACL)

Router(config)# access-list 1 permit 192.168.10.0 0.0.0.255
ACL defines which inside addresses are eligible for NAT (entire 192.168.10.0/24 network)

Step 2: Define Pool of Public IPs

Router(config)# ip nat pool PUBLIC_POOL 203.0.113.10 203.0.113.14 netmask 255.255.255.248
Creates pool named PUBLIC_POOL with 5 usable public IPs (.10 through .14)

Step 3: Link ACL to Pool

Router(config)# ip nat inside source list 1 pool PUBLIC_POOL
Connects ACL 1 (inside addresses) to PUBLIC_POOL. First 5 hosts get public IPs, 6th host blocked until one frees up.

Problem with Dynamic NAT: If pool exhausted (more devices than public IPs), new connections fail. Solution: PAT!

PAT Configuration (NAT Overload)

Most common NAT type. Thousands of internal devices share a single public IP using unique port numbers.

Method 1: PAT using interface IP address (typical home/small office)
Router(config)# access-list 1 permit 192.168.0.0 0.0.255.255
Permits entire 192.168.0.0/16 private network for NAT
Router(config)# ip nat inside source list 1 interface gi0/1 overload
Translates ACL 1 addresses to Gi0/1's public IP. 'overload' keyword enables PAT (port translation).

Example translation: 192.168.1.10:45000 → 203.0.113.1:45000, 192.168.1.20:45001 → 203.0.113.1:45001

Method 2: PAT using pool of addresses (enterprise)
Router(config)# ip nat pool PAT_POOL 203.0.113.1 203.0.113.3 netmask 255.255.255.0
Creates pool of 3 public IPs for PAT
Router(config)# ip nat inside source list 1 pool PAT_POOL overload
Uses pool for PAT. First public IP filled with ~65k translations, then uses next IP in pool.

Port Security

What is Port Security?

Port security restricts which devices can connect to a switch port based on MAC addresses. This prevents unauthorized devices from accessing the network and mitigates MAC flooding attacks. You can limit the number of MACs allowed, specify which MACs are permitted, and define actions when violations occur.

Why Use Port Security?

Prevent Unauthorized Access
Only allowed devices (by MAC) can connect to the port
One ACL per Interface per Direction
Each interface can have one inbound and one outbound ACL. Inbound filters before routing, outbound after.
Inbound vs Outbound
Inbound: Checked as packet enters interface. Outbound: Checked as packet exits interface.

Standard ACLs (1-99, 1300-1999)

Filter based on source IP address only. Simple but limited. Apply close to destination.

Standard ACL Syntax

access-list <1-99> {permit|deny} <source> [wildcard-mask]
Example: Deny host 192.168.10.5, permit all others
Router(config)# access-list 10 deny 192.168.10.5 0.0.0.0
Denies specific host. Wildcard 0.0.0.0 means exact match (can also use keyword "host 192.168.10.5")
Router(config)# access-list 10 permit any
Permits all other traffic. Without this, implicit deny blocks everything else!
Router(config)# interface gi0/0
Enter interface where ACL will be applied
Router(config-if)# ip access-group 10 out
Applies ACL 10 to outbound traffic on this interface. Use 'in' for inbound filtering.

OSPF (Open Shortest Path First)

What is OSPF?

OSPF is a link-state routing protocol that uses Dijkstra's algorithm to calculate the shortest path to destinations. Unlike distance-vector protocols (RIP), OSPF has fast convergence, supports large networks, uses bandwidth as its metric (cost), and sends updates only when topology changes. OSPF is the most common interior gateway protocol (IGP) for enterprise networks.

OSPF Key Concepts

Link-State Protocol
Each router builds complete topology map of network. Routers share link states, not routes.
Cost Metric
Based on bandwidth. Cost = 100,000,000 / bandwidth(bps). Lower cost = better path. FastEthernet cost = 1.
Areas
OSPF divides network into areas for scalability. Area 0 (backbone) is mandatory. All areas must connect to Area 0.
Router ID
Unique identifier for each OSPF router (format: x.x.x.x like IP). Highest loopback IP or highest physical interface IP.
Hello Protocol
Discovers neighbors, maintains adjacencies. Hello every 10 sec (broadcast), dead timer 40 sec. Must match for adjacency.

OSPF Router Types

Internal Router
All interfaces in same area
Backbone Router
At least one interface in Area 0
Area Border Router (ABR)
Connects multiple areas, maintains separate topology database for each area
Autonomous System Boundary Router (ASBR)
Connects OSPF to other routing protocols or external networks (redistributes routes)

Basic OSPF Configuration

Configure OSPF on router with interfaces in 192.168.10.0/24 and 10.1.1.0/24 networks
Router(config)# router ospf 1
Enables OSPF process 1. Process ID is locally significant (doesn't need to match other routers).
Router(config-router)# network 192.168.10.0 0.0.0.255 area 0
Advertises 192.168.10.0/24 network in Area 0. Uses wildcard mask (inverse of subnet mask).
Router(config-router)# network 10.1.1.0 0.0.0.255 area 0
Advertises 10.1.1.0/24 network in Area 0

Setting Router ID

Router ID selection order: 1) Manual configuration, 2) Highest loopback IP, 3) Highest active interface IP

Router(config-router)# router-id 1.1.1.1
Manually sets OSPF router ID to 1.1.1.1. Best practice: use manually for consistency.
Router(config)# interface loopback 0
Create loopback interface (virtual, always up) - OSPF prefers loopback for router ID
Router(config-if)# ip address 1.1.1.1 255.255.255.255
Assign IP to loopback. /32 mask conserves addresses. This becomes router ID if no manual config.

To change router ID after OSPF is running:

Router# clear ip ospf process
Restarts OSPF process to apply new router ID. Causes brief outage - do during maintenance window.

Passive Interfaces

Prevents OSPF hellos on interfaces where no neighbors exist (e.g., user-facing networks). Saves bandwidth and improves security.

Router(config-router)# passive-interface gigabitEthernet 0/0
Disables OSPF hellos on Gi0/0. Network still advertised, but no neighbor adjacencies formed.
Router(config-router)# passive-interface default
Makes all interfaces passive by default. Then enable OSPF only on needed interfaces.
Router(config-router)# no passive-interface gigabitEthernet 0/1
Enables OSPF on Gi0/1 when using passive-interface default

Default Route Advertisement

ASBR can advertise default route (0.0.0.0/0) to all OSPF routers - useful for Internet gateway.

Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
First create static default route to ISP
Router(config-router)# default-information originate
Advertises default route into OSPF. Only works if default route exists in routing table.
Router(config-router)# default-information originate always
Advertises default route even if it's not in routing table. Use with caution!

DR/BDR Election (Multi-Access Networks)

On broadcast networks (Ethernet), OSPF elects Designated Router (DR) and Backup DR to reduce LSA flooding. All routers form adjacency with DR/BDR only.

Election Criteria
1) Highest OSPF priority (default 1), 2) Highest router ID. Priority 0 = never become DR/BDR.
Router(config-if)# ip ospf priority 255
Sets priority to 255 (max) on interface - this router preferred as DR. Range: 0-255.
Router(config-if)# ip ospf priority 0
Priority 0 prevents router from becoming DR/BDR. Use on low-end routers.

Important: DR/BDR election is non-preemptive. Once elected, DR stays until it fails (even if higher priority router joins).

Modifying OSPF Cost

Manipulate path selection by changing interface cost or reference bandwidth.

Router(config-if)# ip ospf cost 50
Manually sets cost to 50 on interface. Overrides automatic calculation.
Router(config-router)# auto-cost reference-bandwidth 10000
Changes reference bandwidth to 10 Gbps (10,000 Mbps). Default is 100 Mbps. Set same on all routers!

Example: With reference 10000, GigabitEthernet cost = 10000/1000 = 10

OSPF Timers

Router(config-if)# ip ospf hello-interval 5
Changes hello interval to 5 seconds (default 10 for broadcast). Must match on both sides!
Router(config-if)# ip ospf dead-interval 20
Changes dead interval to 20 seconds (default 40). Typically 4x hello interval.

Spanning Tree Protocol (STP)

What is Spanning Tree Protocol?

STP prevents Layer 2 loops in switched networks by blocking redundant paths. Without STP, broadcast storms occur when frames loop infinitely through redundant switch connections, causing network crashes. STP ensures one active path between switches while keeping backup paths ready for failover.

Why We Need STP

Broadcast Storms
Loops cause broadcasts to multiply infinitely, consuming all bandwidth within seconds
MAC Table Instability
Switches see same MAC on multiple ports, constantly updating tables, never learning correctly
Multiple Frame Copies
Frames duplicate as they traverse loops, causing hosts to receive multiple copies of same packet

STP Versions

802.1D (STP)
Original STP - convergence ~50 seconds. Per-VLAN support limited. Rarely used today.
PVST+ (Per-VLAN STP)
Cisco proprietary - separate STP instance per VLAN. Convergence ~50 sec. Allows per-VLAN root bridge.
802.1w (RSTP - Rapid STP)
Fast convergence (~6 seconds). Backward compatible with 802.1D. Standard, not per-VLAN.
Rapid PVST+
Cisco proprietary - RSTP with per-VLAN instances. Fast convergence + load balancing. MOST COMMON (default on Cisco).
802.1s (MSTP - Multiple STP)
Maps multiple VLANs to single STP instance. Reduces CPU load. Complex config, less common.

STP Port Roles

Root Port (RP)
Port with lowest cost path to root bridge. One per non-root switch. Forwards traffic.
Designated Port (DP)
Port with best path for segment. One per network segment. Forwards traffic. All root bridge ports are designated.
Alternate Port (RSTP)
Backup to root port. Blocks traffic. Provides fast failover if root port fails.
Backup Port (RSTP)
Backup to designated port on same switch (rare - occurs with hubs or multiple links).
Blocked/Discarding Port
Not root or designated - blocks traffic to prevent loops. Listens to BPDUs.

PortFast

PortFast bypasses listening/learning states - port goes directly to forwarding. ONLY use on access ports connected to end devices (PCs, servers), never on trunk or switch-to-switch links!

Switch(config-if)# spanning-tree portfast
Enables PortFast on interface. Port immediately forwards upon link up. Skips 30-sec delay.
Switch(config)# spanning-tree portfast default
Enables PortFast on all access ports globally. Trunk ports not affected.

Warning: PortFast on trunk/switch links causes loops! Use BPDU Guard to prevent this.

BPDU Guard

Protects against unauthorized switches. If PortFast port receives BPDU (from switch), port shuts down immediately (err-disabled).

Switch(config-if)# spanning-tree bpduguard enable
Enables BPDU Guard on interface. If BPDU received, port goes err-disabled. Use with PortFast.
Switch(config)# spanning-tree portfast bpduguard default
Enables BPDU Guard on all PortFast ports globally. Best practice for security.

BPDU Filter

Suppresses BPDUs on port. Disables STP on interface. Dangerous - use only in specific scenarios (e.g., connecting to devices that don't understand BPDUs).

Switch(config-if)# spanning-tree bpdufilter enable
Stops sending/receiving BPDUs on port. Disables STP entirely. Use with extreme caution!

Root Guard

Prevents port from becoming root port. If superior BPDU received (someone trying to become root), port goes into root-inconsistent state (blocked).

Switch(config-if)# spanning-tree guard root
Prevents unauthorized root bridge on this port. Use on ports facing other switches you don't control.

Loop Guard

Prevents alternate/blocked ports from becoming designated if BPDUs stop arriving (link unidirectional failure).

Switch(config-if)# spanning-tree guard loop
If BPDUs stop, port goes into loop-inconsistent state (blocked) instead of forwarding.
Switch(config)# spanning-tree loopguard default
Enables loop guard globally on all non-PortFast ports

Switch(config)# spanning-tree mode pvst
Sets mode to PVST+ (classic STP, per-VLAN). Convergence ~50 seconds.
Switch(config)# spanning-tree mode rapid-pvst
Sets mode to Rapid PVST+ (default on Cisco). Fast convergence ~6 seconds, per-VLAN. RECOMMENDED.
Switch(config)# spanning-tree mode mst
Sets mode to Multiple Spanning Tree (MSTP). Maps VLANs to instances for efficiency.

Issue: Wrong root bridge elected
Fix: Configure desired switch with 'spanning-tree vlan X root primary' or lower priority manually

Issue: Port flapping between states

IPv6 Basics

What is IPv6?

IPv6 is the next-generation Internet Protocol designed to replace IPv4. With 128-bit addresses (vs IPv4's 32-bit), IPv6 provides 340 undecillion addresses - enough for every device on Earth. IPv6 simplifies address assignment (SLAAC), improves security (IPsec built-in), eliminates NAT requirement, and supports better QoS and mobility.

Why IPv6?

IPv4 Exhaustion
IPv4 has ~4.3 billion addresses - exhausted. IPv6 has 340 undecillion (340 trillion trillion trillion).
No NAT Required
Enough addresses for every device to have global unique IP. NAT complications eliminated.
Simplified Header
IPv6 header streamlined for faster processing. Checksum removed (handled by Layer 2/4).
Built-in Security
IPsec mandatory in IPv6 (optional in IPv4). Authentication and encryption standardized.
Auto-configuration
SLAAC allows devices to self-configure without DHCP server

IPv6 Address Format

IPv6 addresses are 128 bits written as 8 groups of 4 hexadecimal digits separated by colons.

Full Format
2001:0db8:0000:0000:0000:ff00:0042:8329 (128 bits = 32 hex digits)

IPv6 Address Shortening Rules

Rule 1: Drop Leading Zeros
2001:0db8:0000:0000:0000:ff00:0042:8329 → 2001:db8:0:0:0:ff00:42:8329
Rule 2: Replace Consecutive Zeros with ::
2001:db8:0:0:0:ff00:42:8329 → 2001:db8::ff00:42:8329 (use :: only ONCE per address)

Examples

Full: 2001:0db8:0000:0042:0000:0000:0000:0001
Compressed: 2001:db8:0:42::1
Leading zeros removed, consecutive zeros replaced with ::
Full: fe80:0000:0000:0000:0202:b3ff:fe1e:8329
Compressed: fe80::202:b3ff:fe1e:8329
Link-local address - used for local subnet communication

IPv6 Address Types

1. Global Unicast (GUA)

2000::/3 (2000:: to 3fff::)
Public routable addresses - equivalent to IPv4 public IPs. Assigned by ISPs. Most common type.
Example: 2001:db8:acad:1::10/64
Globally unique, routable on Internet. First 64 bits = network prefix, last 64 bits = interface ID (host)

2. Link-Local (LLA)

fe80::/10 (fe80:: to febf::)
Auto-configured on every IPv6 interface. Used for local subnet communication. Not routable beyond local link.
Mandatory
Every IPv6 interface MUST have link-local address. Used for neighbor discovery, DHCPv6, routing protocols.
Example: fe80::1
Common manually configured link-local for routers. Auto-generated uses EUI-64 or random.

3. Unique Local (ULA)

fc00::/7 (fc00:: to fdff::)
Private addresses - equivalent to IPv4 private (10.x, 172.16.x, 192.168.x). Not routable on Internet.
fd00::/8
Locally assigned ULA range (fd = locally assigned). Use for internal networks without Internet routing.

4. Multicast

ff00::/8
Multicast addresses - one-to-many communication. No broadcast in IPv6 (replaced by multicast).
ff02::1
All IPv6 nodes on link (equivalent to IPv4 broadcast 255.255.255.255)
ff02::2
All IPv6 routers on link
ff02::5
All OSPF routers
ff02::6
All OSPF designated routers
ff02::1:ff00:0/104
Solicited-node multicast - used for neighbor discovery (replaces ARP)

5. Special Addresses

::/128 (all zeros)
Unspecified address - equivalent to 0.0.0.0 in IPv4. Used before address assigned.
::1/128
Loopback address - equivalent to 127.0.0.1 in IPv4. Used for local testing (ping ::1)
::ffff:0:0/96
IPv4-mapped IPv6 address - allows IPv6 apps to communicate with IPv4 (e.g., ::ffff:192.168.1.1)
2001:db8::/32
Documentation/example addresses - reserved for documentation (like 192.0.2.0 in IPv4). Never routed.

IPv6 Prefix Length

IPv6 uses prefix length notation (/) - NO subnet masks! Common: /64 for LANs, /48 for sites, /128 for single host.

/64
Standard LAN prefix - 64 bits network, 64 bits host. Allows SLAAC and 18 quintillion hosts per subnet!
/48
Typical enterprise allocation - provides 65,536 /64 subnets (16 bits for subnetting)
/32
ISP allocation to large organizations
/128
Single host (like /32 in IPv4). Use for loopback or point-to-point links.

Enabling IPv6 on Cisco Router

Router(config)# ipv6 unicast-routing
Enables IPv6 routing globally - REQUIRED for router to forward IPv6 packets. Off by default!

Configuring IPv6 Addresses

Method 1: Manual Static Address

Router(config)# interface gi0/0
Enter interface to configure
Router(config-if)# ipv6 address 2001:db8:acad:1::1/64
Assigns static IPv6 address with /64 prefix. Manually configured global unicast address.
Router(config-if)# no shutdown
Enable interface - link-local address automatically generated in addition to configured GUA

Method 2: EUI-64 (Auto-generate Interface ID)

Router(config-if)# ipv6 address 2001:db8:acad:1::/64 eui-64
Uses EUI-64 to auto-generate last 64 bits from MAC address. First 64 bits manually specified.

EUI-64 process: Takes MAC (48 bits) → inserts "fffe" in middle → flips 7th bit → creates 64-bit interface ID

Method 3: SLAAC (Stateless Auto-configuration)

Router(config-if)# ipv6 address autoconfig
Uses SLAAC to learn prefix from router advertisements (RA). Auto-configures address using EUI-64 or privacy extensions.

Configuring Link-Local Address

Router(config-if)# ipv6 address fe80::1 link-local
Manually sets link-local address to fe80::1. Best practice: use simple link-locals (fe80::1, fe80::2, etc.) on routers.

Multiple IPv6 Addresses per Interface

Unlike IPv4, one interface can have multiple IPv6 addresses (one link-local + multiple global unicast).

interface gi0/0
ipv6 address 2001:db8:acad:1::1/64
ipv6 address 2001:db8:cafe:1::1/64
ipv6 address fe80::1 link-local
Interface has 2 global unicast addresses + 1 link-local. Useful for renumbering or multi-homing.

DHCPv6 Configuration

DHCPv6 provides stateful address assignment (like IPv4 DHCP). Two modes: Stateful (DHCPv6 assigns address) and Stateless (SLAAC for address, DHCPv6 for DNS/options).

Stateless DHCPv6 (Common)

Router(config)# ipv6 dhcp pool IPV6-STATELESS
Creates DHCPv6 pool for stateless config (provides DNS, domain, not addresses)
Router(config-dhcpv6)# dns-server 2001:4860:4860::8888
Sets DNS server (Google Public DNS for IPv6)
Router(config-dhcpv6)# domain-name example.com
Sets domain name for clients
Router(config)# interface gi0/0
Enter interface facing clients
Router(config-if)# ipv6 dhcp server IPV6-STATELESS
Applies DHCPv6 pool to interface
Router(config-if)# ipv6 nd other-config-flag
Sets O-flag in RA - tells clients to use SLAAC for address, DHCPv6 for other info (DNS, domain)

IPv6 Static Routes

Router(config)# ipv6 route 2001:db8:acad:2::/64 2001:db8:acad:1::2
Static route using next-hop IPv6 address
Router(config)# ipv6 route 2001:db8:acad:3::/64 gi0/1
Static route using exit interface
Router(config)# ipv6 route ::/0 2001:db8:acad:1::1
Default IPv6 route (equivalent to 0.0.0.0/0) - gateway of last resort
Router(config)# ipv6 route 2001:db8:cafe::/64 gi0/0 fe80::2
Fully specified route - exit interface + link-local next-hop (required for link-local next-hops)

Neighbor Discovery Protocol (NDP)

NDP replaces ARP in IPv6. Uses ICMPv6 messages for address resolution, duplicate detection, router discovery.

Router Solicitation (RS)
Host sends RS (multicast to ff02::2) asking "Any routers here?" upon startup
Router Advertisement (RA)
Router sends RA (multicast to ff02::1) with prefix, gateway info. Sent periodically or in response to RS.
Neighbor Solicitation (NS)
Equivalent to ARP request - "Who has this IPv6 address?" Uses solicited-node multicast.
Neighbor Advertisement (NA)
Equivalent to ARP reply - "I have that address, here's my MAC"
Redirect
Router tells host about better next-hop for destination

Duplicate Address Detection (DAD)

Before using IPv6 address, device sends NS to verify no one else has it (like gratuitous ARP in IPv4).

show ipv6 interface gi0/0
Look for "DAD complete" or "tentative" status. Tentative = DAD in progress.

Common IPv6 Mistakes

Forgetting ipv6 unicast-routing
Router won't forward IPv6 packets without this global command. Must enable explicitly!
Using :: twice in address
Can only use :: once per address. Multiple instances are ambiguous.
Mixing up link-local scope
Link-local (fe80::) only valid on local link. Must specify interface when using as next-hop.
Wrong prefix length
LANs should use /64 for SLAAC compatibility. /127 or /128 for point-to-point.

EtherChannel

What is EtherChannel?

EtherChannel (also called Link Aggregation or Port Channel) bundles multiple physical links into one logical link. This increases bandwidth, provides redundancy, and prevents STP from blocking redundant links. Instead of 1 Gbps, you can have 2, 4, or 8 Gbps between switches. EtherChannel appears as a single interface to STP, preventing loops while allowing full use of all links.

Why Use EtherChannel?

Increased Bandwidth
Combine multiple links (2x 1Gbps = 2Gbps total). Up to 8 physical links per bundle.
Load Balancing
Traffic distributed across all active links based on MAC, IP, or port. No single link overload.
Redundancy
If one link fails, traffic automatically shifts to remaining links. No STP convergence delay!
STP Friendly
STP sees EtherChannel as single link, won't block redundant paths. All links forward simultaneously.

EtherChannel Protocols

PAgP (Port Aggregation Protocol)
Cisco proprietary. Negotiates EtherChannel formation. Modes: desirable, auto. Not compatible with non-Cisco.
LACP (Link Aggregation Control Protocol)
IEEE 802.3ad standard. Open standard, works with any vendor. Modes: active, passive. RECOMMENDED.
Static (On Mode)
No negotiation protocol. Manually forces EtherChannel. Risky - misconfig causes loops. Not recommended.

EtherChannel Requirements

All interfaces in EtherChannel MUST match these settings or EtherChannel fails:

✓ Same Speed
All ports must be same speed (all 1Gbps or all 10Gbps). Can't mix 100Mbps with 1Gbps.
✓ Same Duplex
All ports must be full-duplex (or all half-duplex). Full-duplex is standard.
✓ Same VLAN (Access Mode)
If access ports, all must be in same VLAN. Can't bundle ports from different VLANs.
✓ Same Trunk Settings (Trunk Mode)
If trunk, must have same native VLAN and allowed VLANs. Encapsulation must match (802.1Q).
✓ Same Mode (Access or Trunk)
All ports must be access OR all trunk. Can't mix access with trunk.

Basic EtherChannel Configuration (LACP)

Create Layer 2 EtherChannel between two switches using Fa0/1 and Fa0/2

On Switch 1

Switch1(config)# interface range fa0/1-2
Select both interfaces to bundle. Can use range for efficiency.
Switch1(config-if-range)# channel-group 1 mode active
Assigns interfaces to channel-group 1 using LACP active mode. Number (1) is locally significant.
Switch1(config-if-range)# exit
Exit interface config. Port-Channel1 interface automatically created.

On Switch 2

Switch2(config)# interface range fa0/1-2
Select matching interfaces on second switch
Switch2(config-if-range)# channel-group 1 mode active
Same channel-group number and mode. LACP active + active = EtherChannel forms!

Configuring Port-Channel Interface

After physical interfaces bundled, configure the logical Port-Channel interface for VLAN/trunk settings.

Configure EtherChannel as trunk carrying VLANs 10, 20, 30
Switch(config)# interface port-channel 1
Enter Port-Channel interface. Number matches channel-group number.
Switch(config-if)# switchport mode trunk
Configure as trunk. Applied to ALL member interfaces automatically.
Switch(config-if)# switchport trunk allowed vlan 10,20,30
Limit allowed VLANs. Configuration propagates to all physical ports in bundle.
Switch(config-if)# switchport trunk native vlan 99
Sets native VLAN. Must match on both ends of EtherChannel.

EtherChannel can also be Layer 3 (routed) instead of Layer 2 (switched). Use between routers or Layer 3 switches.

Router(config)# interface range gi0/0-1
Select physical interfaces
Router(config-if-range)# no switchport

Router(config-if)# no switchport

Load Balancing Methods

EtherChannel distributes traffic across links using hashing algorithms. Default varies by platform.

src-mac
Load balance based on source MAC address. Simple, but can be uneven.
dst-mac
Load balance based on destination MAC address
src-dst-mac
Uses both source and destination MAC. Better distribution. DEFAULT on many switches.
src-ip
Load balance based on source IP address
dst-ip
Load balance based on destination IP address
src-dst-ip
Uses both source and destination IP. Good for Layer 3 traffic. RECOMMENDED for routed EtherChannel.
src-dst-port
Uses source and destination TCP/UDP ports. Best granularity, most even distribution.

Configuring Load Balancing

Switch(config)# port-channel load-balance src-dst-ip
Sets load balance method GLOBALLY (affects all EtherChannels). Use src-dst-ip for IP traffic.
Switch# show etherchannel load-balance
Shows current load balancing method in use

Troubleshooting EtherChannel

Issue: Ports show (I) instead of (P)
Cause: EtherChannel not formed. Ports in stand-alone mode. Check: mode mismatch, config mismatch, both sides configured?
Issue: Ports show (D) - down
Cause: Physical ports down. Check: cables connected? Ports enabled (no shutdown)? Link lights?
Issue: Ports show (s) - suspended
Cause: Configuration mismatch. Check: speed, duplex, VLAN, trunk settings all match?
Issue: EtherChannel forms but no traffic
Check: Port-Channel interface up? Trunk allowed VLANs correct? Native VLAN matches? Load balance method appropriate?
Issue: Uneven traffic distribution
Cause: Load balance method. Fix: Change to src-dst-ip or src-dst-port for better distribution.
Issue: % Port not compatible
Error when adding port to channel-group. Cause: Port config doesn't match other members. Fix: Match speed, duplex, VLAN.

Common Configuration Mismatches

Speed/Duplex Mismatch
One port 100Mbps, another 1Gbps = EtherChannel fails. All must match!
VLAN Mismatch
Gi0/1 in VLAN 10, Gi0/2 in VLAN 20 = EtherChannel fails. All access ports must be same VLAN.
Trunk Native VLAN Mismatch
Native VLAN differs between ports = EtherChannel forms but CDP warnings. Fix: match native VLAN.
Mode Mismatch (Access vs Trunk)
One port access, another trunk = EtherChannel fails. All must be same mode.
Protocol Mismatch
One side LACP, other side PAgP = EtherChannel fails. Must use same protocol on both ends.

Modifying EtherChannel

Best practice: Make config changes on Port-Channel interface, not physical interfaces.

✓ Configure on Port-Channel
Changes automatically apply to all member ports. Consistent configuration guaranteed.
✗ Don't configure physical ports directly
Causes inconsistencies. Port may be suspended from channel. Always use Port-Channel interface!

Adding/Removing Ports from EtherChannel

Add Gi0/3 to existing EtherChannel 1
Switch(config)# interface gi0/3
Enter new interface to add
Switch(config-if)# channel-group 1 mode active
Adds to existing channel-group 1. Must match protocol/mode. Port inherits Port-Channel config.
Remove Gi0/2 from EtherChannel
Switch(config)# interface gi0/2
Enter interface to remove
Switch(config-if)# no channel-group 1
Removes port from EtherChannel. Port becomes stand-alone. Traffic redistributes to remaining ports.

Deleting EtherChannel

Switch(config)# no interface port-channel 1
Deletes Port-Channel interface. Physical ports remain but become stand-alone.

Or remove channel-group from all physical interfaces - Port-Channel automatically deleted when last member removed.

EtherChannel and STP

STP treats EtherChannel as single logical link. Port-Channel has one role (root/designated/alternate), not per physical port.

STP Configuration
Apply STP settings (cost, priority, PortFast) to Port-Channel interface, not physical ports
interface port-channel 1
spanning-tree vlan 10 cost 10
Sets STP cost for entire EtherChannel, not individual links

LACP Priority (Advanced)

If more than 8 ports in channel-group, LACP uses priority to select which ports are active (max 8 active).

Switch(config-if)# lacp port-priority 100
Sets LACP port priority (default 32768). Lower = more preferred. Range: 0-65535.
Switch(config)# lacp system-priority 100
Sets system-wide LACP priority. Determines which switch is "active" in negotiations.

Best Practices

Use LACP over PAgP
LACP is open standard, works with all vendors. PAgP is Cisco proprietary.
Use active mode on both sides
Active + active ensures immediate negotiation. Avoids passive + passive failure.
Avoid "on" mode
Static mode (on) has no safeguards. Use LACP/PAgP for negotiation and error detection.
Configure Port-Channel interface, not physical
All settings (VLAN, trunk, STP) on Port-Channel. Ensures consistency across members.
Use 2, 4, or 8 ports
Powers of 2 provide best load distribution. Max 8 active ports per EtherChannel.
Use src-dst-ip or src-dst-port load balance
Best distribution for most networks. Src-dst-port ideal for varied traffic.

EIGRP

What is EIGRP?

EIGRP (Enhanced Interior Gateway Routing Protocol) is a Cisco advanced distance-vector routing protocol that combines the best of distance-vector and link-state protocols. It features fast convergence, low bandwidth usage (sends partial updates only), support for VLSM/CIDR, automatic route summarization, and load balancing across unequal-cost paths. EIGRP uses DUAL (Diffusing Update Algorithm) to guarantee loop-free paths.

EIGRP Key Features

Hybrid Protocol
Combines best of distance-vector (simple) and link-state (fast convergence). Cisco calls it "advanced distance-vector".
Fast Convergence
Uses DUAL algorithm to find backup routes instantly. No need to recalculate entire topology on failure.
Partial Updates
Only sends updates when topology changes, and only changes (not entire routing table). Saves bandwidth.
Unequal-Cost Load Balancing
Can load balance across paths with different metrics using variance. Unique to EIGRP!
VLSM Support
Supports variable-length subnet masks and CIDR. Classless routing protocol.
Multiple Network Layer Support
Originally for IPv4, now supports IPv6, IPX, AppleTalk (legacy). IPv6 EIGRP configured separately.

EIGRP Terminology

Autonomous System (AS)
Group of routers under same administrative control. AS numbers must match for neighbor adjacency. Range: 1-65535.
Neighbor Table
List of directly connected EIGRP routers. Shows neighbor IP, interface, hold time. View: show ip eigrp neighbors
Topology Table
Contains all routes learned from neighbors - both best and backup routes. View: show ip eigrp topology
Routing Table
Contains best routes selected from topology table. These routes actually used for forwarding. View: show ip route eigrp
Successor
Best route to destination (lowest metric). Installed in routing table. Used for forwarding traffic.
Feasible Successor (FS)
Backup route that meets feasibility condition. Pre-calculated, instantly used if successor fails. Loop-free backup!
Feasible Distance (FD)
Metric of best path to destination (through successor). Lowest total cost from this router.
Reported Distance (RD) / Advertised Distance
Neighbor's metric to destination. What neighbor reports as its FD. Used to check feasibility condition.

Feasibility Condition

For a route to be feasible successor (backup), its Reported Distance must be LESS than the current Feasible Distance. This guarantees loop-free path.

Feasibility Condition:
Reported Distance (RD) < Feasible Distance (FD)
If neighbor's cost to destination is less than my current best cost, it's a valid backup (guaranteed loop-free)

EIGRP Metric Calculation

EIGRP uses composite metric based on bandwidth, delay, load, reliability, and MTU. By default, only bandwidth and delay are used.

K-Values (Metric Weights)
K1=bandwidth, K2=load, K3=delay, K4/K5=reliability. Default: K1=1, K3=1, others=0. Must match for adjacency!
Default Metric Formula (K1=1, K3=1, others=0):
Metric = (K1 × Bandwidth + K3 × Delay) × 256
Bandwidth = (10^7 / slowest link in kbps)
Delay = sum of all delays in tens of microseconds
Bandwidth (Primary)
10,000,000 / slowest bandwidth in path (in kbps). Lower bandwidth = higher metric (worse).
Delay (Primary)
Cumulative delay in tens of microseconds. Sum of all link delays in path.

Example: FastEthernet (100Mbps, 100μs delay): Metric = [(10^7/100000) + (100/10)] × 256 = 28,160

EIGRP Packet Types

Hello
Discovers neighbors, maintains adjacencies. Multicast (224.0.0.10). Interval: 5 sec (LAN), 60 sec (WAN).
Update
Sends routing information. Only sent when changes occur (partial updates). Uses RTP for reliability.
Query
Sent when route is lost and no feasible successor exists. Asks neighbors "Do you have path to this network?"
Reply
Response to Query. Answers with metric to destination or "unreachable" if no path.
ACK
Acknowledges reliable packets (Update, Query, Reply). Unicast. Ensures delivery.

Network Statement - Classful vs Wildcard

Two ways to specify networks: classful (simple) or with wildcard mask (precise).

network 10.0.0.0
Classful - advertises entire Class A (10.0.0.0/8). Simple but broad.
network 192.168.10.0 0.0.0.255
Wildcard - advertises specific /24 subnet. More precise control.
network 192.168.10.1 0.0.0.0
Match specific interface IP exactly (host route). Use for single interface.

Router ID

Router ID selection: 1) Manual configuration, 2) Highest loopback IP, 3) Highest active interface IP

Router(config-router)# eigrp router-id 1.1.1.1
Manually sets EIGRP router ID. Best practice for consistency. Format: x.x.x.x (like IP address).
Router# clear ip eigrp neighbors
Restarts EIGRP process to apply new router ID. Causes brief outage - do during maintenance.

Passive Interface

Prevents EIGRP hellos on interfaces where no neighbors exist. Network still advertised, but no adjacencies formed.

Router(config-router)# passive-interface gigabitEthernet 0/0
Stops EIGRP hellos on Gi0/0. Use on user-facing interfaces (LAN segments with no routers).
Router(config-router)# passive-interface default
Makes all interfaces passive by default. Then explicitly enable EIGRP only on needed interfaces.
Router(config-router)# no passive-interface gigabitEthernet 0/1
Enables EIGRP on Gi0/1 when using passive-interface default

Default Route Propagation

Advertise default route to all EIGRP neighbors (e.g., for Internet gateway)
Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1
First, create static default route pointing to ISP
Router(config-router)# redistribute static

Router(config-router)# network 0.0.0.0

EIGRP Timers

Hello Interval
How often hellos sent. Default: 5 sec (LAN/high-speed), 60 sec (WAN/low-speed T1 and slower)
Hold Timer
How long to wait before declaring neighbor dead. Default: 3× hello interval (15 sec LAN, 180 sec WAN)
Router(config-if)# ip hello-interval eigrp 100 10
Changes hello interval to 10 seconds for AS 100 on this interface
Router(config-if)# ip hold-time eigrp 100 30
Changes hold time to 30 seconds for AS 100. Typically set to 3× hello.

Note: Timers don't need to match for adjacency (unlike OSPF), but best practice is to match them.

Bandwidth and Delay Tuning

Modify bandwidth or delay to influence EIGRP path selection without changing actual link speed.

Router(config-if)# bandwidth 1000
Sets bandwidth to 1000 kbps for metric calculation. Doesn't change actual speed, only EIGRP metric.
Router(config-if)# delay 100
Sets delay to 100 (tens of microseconds = 1000 μs = 1 ms). Affects EIGRP metric calculation.

Tip: Prefer changing delay over bandwidth (bandwidth affects other protocols like QoS).

Load Balancing

Equal-Cost Load Balancing (Default)

Router(config-router)# maximum-paths 4
Sets max equal-cost paths to 4 (default). EIGRP load balances across up to 4 equal-metric routes. Range: 1-32.

Unequal-Cost Load Balancing (Variance)

Unique to EIGRP! Load balance across paths with different metrics using variance multiplier.

Router(config-router)# variance 2
Allows load balancing with routes up to 2× best metric. If best metric = 1000, uses routes up to 2000.
Example:
Best route metric: 1000 (installed)
Alternate route metric: 1500 (feasible successor)
variance 2: Both routes used (1500 < 1000 × 2)
variance 1: Only best route used (1500 > 1000 × 1)

Important: Variance only affects feasible successors (routes meeting feasibility condition). Not all backup routes!

Manual Route Summarization

Summarize multiple networks into single route to reduce routing table size and update traffic.

Summarize networks 172.16.0.0/24 through 172.16.3.0/24 as 172.16.0.0/22
Router(config)# interface gi0/0
Enter interface where summary will be advertised
Router(config-if)# ip summary-address eigrp 100 172.16.0.0 255.255.252.0
Advertises summary route 172.16.0.0/22 instead of individual /24 routes. AS 100 must match.

Summary automatically creates null0 route locally to prevent routing loops.

Automatic Summarization (Legacy)

Router(config-router)# auto-summary
Enables automatic summarization at classful boundaries. DEFAULT in older IOS. Generally should be DISABLED!
Router(config-router)# no auto-summary
Disables auto-summary. REQUIRED for modern networks with VLSM/CIDR. Always use this!

Warning: Auto-summary causes issues with discontiguous networks. Always disable unless you have specific reason!

Authentication

Secure EIGRP with MD5 authentication to prevent rogue routers from injecting routes.

Step 1: Create Key Chain

Router(config)# key chain EIGRP_KEYS
Creates key chain named EIGRP_KEYS
Router(config-keychain)# key 1
Creates key ID 1. Can have multiple keys for key rotation.
Router(config-keychain-key)# key-string MySecretPassword
Sets password for this key. Must match on both sides! Case-sensitive.

Step 2: Apply to Interface

Router(config)# interface gi0/0
Enter interface facing neighbor
Router(config-if)# ip authentication mode eigrp 100 md5
Enables MD5 authentication for EIGRP AS 100 on this interface
Router(config-if)# ip authentication key-chain eigrp 100 EIGRP_KEYS
Applies key chain EIGRP_KEYS to this interface. Must match on both routers!

Troubleshooting EIGRP Adjacencies

Issue: No neighbors forming
Check: 1) AS numbers match? 2) Network statements correct? 3) Interfaces up? 4) Authentication matching? 5) Interface passive?
Issue: Neighbor stuck in pending/initializing
Cause: Not receiving updates or ACKs. Check: MTU mismatch? ACL blocking multicast 224.0.0.10? Firewall?
Issue: Neighbor flapping (up/down)
Check: Hold timer expiring (show ip eigrp neighbors - watch hold time). Interface errors? Duplex mismatch?
Issue: Routes not in routing table
Check: In topology table? (show ip eigrp topology). If yes but not in routing table, check AD or duplicate from other protocol.
Issue: K-values mismatch
Error: "Not on common subnet" or K-values differ. Fix: K-values MUST match for adjacency (default K1=1, K3=1, others=0).

EIGRP Route States

Passive (P)
Route stable, no recalculation needed. Normal operational state. Show as "P" in topology table.
Active (A)
Route lost, router querying neighbors for alternate path. Should be brief. Show as "A" in topology table.
Stuck-in-Active (SIA)
Route active for 3+ minutes (no reply to queries). Neighbor relationship torn down. Indicates network problem!

DUAL Finite State Machine

DUAL ensures loop-free operation and fast convergence. When successor fails:

If Feasible Successor exists
Instantly promoted to successor. No queries sent. Convergence in milliseconds!
If NO Feasible Successor
Route goes Active. Sends Queries to all neighbors. Waits for Replies. Calculates new route. Slower convergence.

Best Practices

Always use "no auto-summary"
Prevents issues with discontiguous networks and VLSM. Required for modern networks.
Set router ID manually
Use "eigrp router-id" for consistency and predictability
Use passive-interface on LANs
Prevents unnecessary EIGRP traffic on user-facing segments. Saves bandwidth and improves security.

Use MD5 authentication on WAN links to prevent rogue routers. Security best practice.

Reduce routing table size and query scope. Improves stability and convergence.
Tune delay, not bandwidth
Bandwidth affects QoS. Delay only affects EIGRP metric. Safer to modify.
