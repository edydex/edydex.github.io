Interface Configuration

*interface <type> <number>
Enters interface configuration mode
*ip address <ip> <subnet-mask>
Assigns IP address and subnet mask to interface
no shutdown
Enables the interface
shutdown
Disables the interface
description <text>
Adds description to interface
duplex {auto|full|half}
Sets duplex mode for interface
speed {auto|10|100|1000}
Sets interface speed

VLAN Configuration

*vlan <vlan-id>
Creates VLAN and enters VLAN config mode
*name <vlan-name>
Names the VLAN (in VLAN config mode)
*switchport mode access
Sets port as access port
*switchport access vlan <vlan-id>
Assigns access port to VLAN
*switchport mode trunk
Sets port as trunk port
*switchport trunk allowed vlan {add|remove|all|except} <vlan-list>
Specifies allowed VLANs on trunk
*switchport trunk native vlan <vlan-id>
Sets native VLAN for trunk port

Port Security

*switchport port-security
Enables port security on interface
*switchport port-security maximum <num>
Sets max number of allowed MAC addresses
*switchport port-security mac-address {<mac>|sticky}
Manually sets MAC or enables sticky learning
*switchport port-security violation {shutdown|restrict|protect}
Sets action when violation occurs

Common Scenarios

Port Security: Restrict Mode with Sticky Learning
Configure port security to allow first 2 MAC addresses, remember them, drop additional packets, and increment violation count
interface FastEthernet0/1
Navigate to the interface connecting to PC2 or server - adjust interface type/number based on your topology
switchport mode access
Port security requires the interface to be in access mode (not trunk)
switchport port-security
Enables port security on the interface - this must be configured before other port-security commands
switchport port-security maximum 2
Allows exactly 2 MAC addresses on this port - matches the "first two devices" requirement
switchport port-security mac-address sticky
Dynamically learns MACs and adds them to running-config - this is how the port "remembers" the first 2 devices
switchport port-security violation restrict
Drops packets from unauthorized MACs AND increments violation counter (unlike protect mode which doesn't count)
show port-security interface FastEthernet0/1
Verify configuration shows Max: 2, Violation Mode: Restrict, and learned MAC addresses
Port Security: Protect Mode (No Violation Count)
Configure port security to allow first 2 MACs, drop additional packets, but NOT increment security-violation count
interface FastEthernet0/2
Navigate to the interface - commonly the PC1 interface in exam scenarios
switchport mode access
Port security only works on access ports
switchport port-security
Enables port security - must come before other port-security commands
switchport port-security maximum 2
Sets max allowed MACs to 2 - this allows "first two devices" to connect
switchport port-security violation protect
Silently drops packets from unauthorized MACs without incrementing violation counter - key difference from restrict mode
show port-security interface FastEthernet0/2
Verify Violation Mode shows "Protect" and Security Violation count stays at 0 even when violations occur
Check Port State for Connected Device
Determine the operational state of a port connected to a specific device (e.g., Inventory server)
show running-config
First, identify which interface has a description matching the device (e.g., "Inventory server") or check the topology
show ip interface brief
Quick view of all interfaces - Status column shows Layer 1 (up/down), Protocol shows Layer 2 (up/down)
show interfaces FastEthernet0/3
Detailed interface info - look for line protocol status and any error counters or shutdown state
show interfaces status
Shows port name, status (connected/notconnect/disabled), VLAN, duplex, speed - easier to read than full interface output
show port-security interface FastEthernet0/3
If port security is configured, check if port is in secure-down or secure-shutdown state due to violations
Basic Router on a Stick (Inter-VLAN Routing)
Configure a router to route between VLANs 10, 20, and 30 using a single trunk link
interface GigabitEthernet0/0
Enter the physical interface that connects to the switch trunk port
no shutdown
Enable the physical interface - subinterfaces inherit this state
interface GigabitEthernet0/0.10
Create subinterface for VLAN 10 - subinterface number typically matches VLAN ID for clarity
encapsulation dot1q 10
Tag traffic with VLAN 10 using 802.1Q - this must match the VLAN on the switch
ip address 192.168.10.1 255.255.255.0
Assign IP as default gateway for VLAN 10 devices - typically use .1 or .254 in the subnet
interface GigabitEthernet0/0.20
Create subinterface for VLAN 20
encapsulation dot1q 20
Tag traffic with VLAN 20
ip address 192.168.20.1 255.255.255.0
Assign default gateway IP for VLAN 20
show ip interface brief
Verify all subinterfaces are up/up and have correct IPs assigned
